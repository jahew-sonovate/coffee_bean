Welcome to Coffee Bean!

A playground for front end developers at Sonovate

Coffee Bean is a simple Node / Angular application that allows front end developers to tune their skills with angular.js and general front end development. The idea with this app is to provide a back end API, which can be used to build a front end for it.

Coffee is an important part of any developer's tool belt. We at Sonovate enjoy a nice cup of joe, and like to know about the coffee we drink. This application should allow us to view a list of our favourite coffee and then view details of each coffee.

Instructions

Using the API as described below, we want you to build an angular.js SPA that will provide two views:

  • A list view - to display a list of all the coffee. There are only 5 coffees to view, so there is no need to think about pagination.
  • A detail view - to display the details for a particular coffee

Your application should make use of either angular's own routing mechanism or bootstrap's UI framework. You should provide reasons for your choice. You are completely free to install any number of bower components you need. If your are not familiar with bower, I recommend you take a look at the website.

For details on how to submit your work, please check the GitHub Repository

The API

Below is a description of the API endpoints you can use in your angular application. All data for this application has been taken from Pact Coffee

Endpoint Description
GET /api/coffee Lists all the coffee we like
GET /api/coffee/:slug Gets the detail of the given coffee. For example, to get details for the Planalto coffee you would request: GET /api/coffee/planalto

The Model

Each coffee data item contains the following properties:

Property Name Description
name The name of the coffee
slug The URL friendly identifier for the coffee. This is a unique value for each coffee.
flavour A brief description of the coffee's flavour
description A description of how the coffee tastes.
country The country of origin
producer The company / person who made the coffee

Responses

The API provides data in JSON format. Each response will contain a result object which will contain the data requested.

Example

A request for the Planalto coffee would return a 200 OK response that looks like this:

{
  "result": {
	"name": "Planalto",
	"slug": "planalto",
	"flavour": "Malt Milk Chocolate Shake",
	"description": "Rich chocolate and malt flavo...",
	"country": "Brazil",
	"producer": "Mariza Aparecida Franco Mantovanini and Sergio de Campos Mantovaninni"
  }
}

While a list request would return a result that contained an array:

{
  "result": [
    {...},
    {...},
  ]
}
